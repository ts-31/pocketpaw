## Overview

Triggers a full health check run including both startup checks (config, storage) and connectivity checks (LLM reachability). Returns the updated health summary.

Unlike [`GET /api/health`](/api/get-health-status) which returns cached results, this endpoint actively runs all 11 checks and updates the health engine state. After completion, it broadcasts the updated health status to all connected WebSocket clients as a `health_update` message.

<Callout type="info">
Connectivity checks make network calls with a 5-second timeout (e.g. testing the Anthropic API). The response may take a few seconds.
</Callout>

## Request Body

No request body is required.

## Response

<ResponseField name="status" type="string">
  Updated overall system health: `healthy`, `degraded`, or `unhealthy`.
</ResponseField>

<ResponseField name="check_count" type="integer">
  Total number of health checks that were run.
</ResponseField>

<ResponseField name="issues" type="array">
  List of check results that are not `ok`. Same format as [`GET /api/health`](/api/get-health-status).
</ResponseField>

<ResponseField name="last_check" type="string">
  ISO 8601 timestamp of this check run.
</ResponseField>
